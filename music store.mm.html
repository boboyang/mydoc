    • + - atool
      • ~ setup
      • instruction

        • 1)            Instrument code: (...\group\): atool -me -acs 5 -fcs 5 sbs -c armv5_urel

          2)            Create urel sis and install it

          3)            Start capturing RDebug logs

          4)            Do something you need to analyze (e.g. start videoplayer, do something that leaks and exit videoplayer)

          5)            Save the RDebug log to ASCII-file (...\group\trace.txt)

          6)            Parse: (…\group\): atool -p trace.txt data.dat

          7)            Analyze: (…\group\): atool -a data.dat memleaks.txt

                          Check call stacks of allocs to leaked mem from “memleaks.txt”.  Call stacks may not always be very accurate. Also, “-acs 5” -value may need to be increased to get longer call stacks.

          8)            Clean instrumentation: (…\group\): atool –c

          P.S. Execution of instrumented code gets often very slow and “ICL” threads will crash many times on startup, but just ignore those ;) For some reason, I have got only “Version: 1.9.0” (atool -v) to work ok so far, other versions may print no trace output needed for analysis.
